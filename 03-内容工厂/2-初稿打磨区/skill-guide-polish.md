# 普通人怎么玩转 AI？从这个词开始

大家好，我是老里。

最近一个词在 AI 圈越来越火：**Skill**。

GitHub 上相关项目星标暴涨，Anthropic 官方亲自维护技能仓库，技能商店 skills.sh 都有 60K+ 订阅量了。

但身边大多数人看到这个词的第一反应是：这是啥？跟我有啥关系？

今天聊聊这个。

---

## 先说个故事

你刚招了一个新员工。

每次让他做事，都得从头教一遍：这个报告要先去财务室拿数据，然后用这个表格汇总，最后发给我审核……

第二天又让他做事，又得说一遍。

累不累？

现在换一种做法。你把流程写成一份《员工手册》，从此以后他看到任务就知道该怎么干，不用每次解释。

**这份员工手册，就是 Skill。**

说人话：Skill = 给 AI 的"操作手册"。

你把工作流程、标准、注意事项写清楚，AI 就会按你的方式来，不用每次重新解释。

---

## 为什么普通人也需要？

ChatGPT 出来两年了，大多数人的用法还是："帮我写个总结""帮我翻译一下"。

问一句答一句，就像个高级搜索引擎。

原因很简单——每次都要重新解释你的需求，AI 根本不知道你想要什么风格的输出。

Skill 解决的是这个根本问题。

几个真实案例，都是完全不会编程的人：

- 做了18年的环评工程师，做了一个"环评报告自动生成"Skill，效率提升10倍
- 产品经理做了一个"天然气数据日报"Skill，从30分钟压到1分钟
- 日语小白做了一个"日本旅行社交指南"Skill，翻译软件做不到的它能做到

这些人都懂自己的业务，只是把业务逻辑告诉了 AI。

---

## 怎么做一个自己的 Skill？

四种方法，总有一种适合你：

**最简单的：直接让 AI 帮你做**

```
我想做一个"每日资讯摘要"的Skill，请你帮我设计一下。
```

AI 会帮你调研方案，确认了直接开发。

**做完一次后保存**

你和 AI 完成了一次不错的协作，直接说："把这个流程保存成 Skill，下次要用。"

**把文章丢给 AI**

你看到一篇讲工作流程的文章，直接复制给 AI："根据这篇文章帮我生成一个 Skill。"

**用官方工具**

```
帮我安装 skill-creator
/skill-creator
```

它会一步步引导你完成。

---

## 举个具体例子

假设你是 HR，经常要写员工入职欢迎邮件。

做一个 Skill：

```
name: welcome-email
描述: 生成员工入职欢迎邮件

流程:
1. 确认姓名、部门、入职日期
2. 生成欢迎语（温暖、专业）
3. 列出入职当天安排
4. 说明首周任务
5. 附团队成员介绍

风格: 温暖不油腻，300-500字
```

以后只需要说：`/welcome-email 张三，技术部，3月5日入职`

AI 自动出完整邮件。一次配置，永久生效。

## 进阶：多个 AI 一起干活

有人会问：一个 AI 干活，能不能多找几个 AI 配合？

这就是 **Agent Team**。

就像公司里有策划、执行、审核，大家配合着来。

以写文章为例：

- **Writer Agent**（擅长写作）负责写初稿
- **Reviewer Agent**（擅长审核）负责打分 + 提意见

写→审核→反馈→重写→再审核→通过，最多循环3轮，质量比从前好很多。

不同 Agent 还可以用不同模型：写作用创意强的，审核用分析强的。

## 几个常见误区

**Skill 不等于提示词。**

提示词是一次性指令，用完就没了。Skill 是持久化的工作方式，一直生效。

**Skill 不需要写代码。**

只要把你做事的流程用大白话写出来就行。

**Agent Team 不是所有场景都适用。**

多个 AI 协作有协调成本，复杂任务才值得用。简单问题反而浪费。

## 现在就能做的事

第一步，安装 find-skills，用自然语言搜索社区里现成的 Skill：

```bash
npx skills add vercel-labs/agent-skills@find-skills -g -y
```

第二步，找几个有趣的 Skill 试试：

- Remotion — 用代码做视频
- frontend-design — 让 AI 生成的界面变好看
- frontend-slides — 用自然话做 PPT

第三步，做一个自己的 Skill。

就从你最熟悉的工作流程开始，不用多复杂，先体验一次"一次配置，永久生效"的感觉。
