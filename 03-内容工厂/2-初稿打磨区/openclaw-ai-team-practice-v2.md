## 一、为什么我要搭建「AI 团队」？

在说具体怎么搭建之前，我想先聊聊为什么。

### 1.1 一个人的困境

做自媒体的人都知道，内容生产是最大的瓶颈。

写一篇公众号文章，从选题、查资料、写作、排版、配图、发布，一套流程下来没三四个小时根本搞不定。而且很多工作都是重复的：每次写文章都要解释一遍自己的写作风格，每次发 Twitter 都要重新组织语言，每次找选题都要从零开始刷新闻。

我就一个人，精力有限。如果什么都自己干，永远被困在「生产内容」这个泥潭里，没有时间思考更重要的事：**做什么内容、怎么做得更好、怎么变现。**

### 1.2 真正的需求是什么？

我想了很久，我发现我真正需要的不是一个 AI，而是一个**团队**。

一个团队意味着：
- **有人帮我做我不想做的事**（比如每天刷新闻找选题）
- **有人帮我做我做不好的事**（比如排版、配图）
- **有人能跟我协作**（我写完公众号，Twitter 那边能自动接上）

这才是我对 AI 的真正期望。不是问一句答一句的工具，而是一个能帮我分担工作的「人」。

### 1.3 为什么不直接买服务？

市场上有很多 AI 服务，订阅制、按量付费的一大把。但我最终还是选择了 OpenClaw，原因是：

**可控。**

我的写作风格、用户偏好、踩过的坑，这些敏感信息我不想放到第三方服务上。OpenClaw 是本地部署的，数据在自己手里，安全感完全不一样。

**可扩展。**

现在我只用了 4 个 AI 角色，但以后想加随时可以加。写代码的、做视频的、搞数据分析的，都可以慢慢加进来。

**可定制。**

很多 AI 服务是「标准化」的，你只能用他们设定好的方式。但 OpenClaw 可以完全按我的需求来，想怎么调就怎么调。

## 二、我的「AI 团队」是怎么分工的？

现在我的团队里有 4 个人，分工明确，各司其职。

### 2.1 团队一览

| AI 角色 | 职责 | 用什么模型 | 怎么配合 |
|---------|------|-----------|---------|
| **小虾米**（我助理） | 统筹、记忆、对外沟通 | MiniMax M2.5 | 有什么事找我，统一调度所有 AI |
| **内容总监** | 每天早上推选题、筛新闻 | MiniMax M2.5 | 9 点准时推送 10-15 个选题 |
| **李笔** | 写 Twitter | MiniMax M2.5 | 我写完公众号，他帮我改写成 Twitter 线程 |
| **AI 工程师** | 写代码、调试脚本 | 智谱 GLM-4-Coding | 我搞不定技术问题时找他 |

### 2.2 每个人详细介绍

**小虾米——我的统筹大脑**

这是我的主账号，也是唯一跟我直接对话的 AI。

他的核心任务不是具体干活，而是**记住我的一切、协调其他 AI**。

他记住我的：
- 写作风格（喜欢用自己的故事切入，不喜欢说教腔）
- 用户画像（中小学教师、AI 爱好者）
- 踩过的坑（发错过账号、封面太复杂导致超时）
- 偏好习惯（喜欢简洁的封面提示词）

当我想让其他 AI 干活时，都先跟小虾米说，由他去调度。

比如：
> 「让李笔帮我把这篇公众号改成 Twitter 线程」
> 
> 小虾米就会调用 libi agent，把任务发过去。

**内容总监——我的选题机器**

这是我设的第一个「业务 AI」。

每天早上 9 点，他会自动运行，从各种 AI 新闻源筛选 10-15 个值得写的选题，推送给我。我只需要挑 3 个开始写就行。

这个有多爽？

以前我每天早上要花 30 分钟以上刷新闻、筛选选题。现在这笔时间全省了。

**李笔——我的 Twitter 运营**

公众号和 Twitter 的风格完全不一样。公众号可以写长文，Twitter 要短、平、快，而且要用很多话题标签。

以前每次发完公众号，我都要手动把内容改写成 Twitter 格式，累死了。

现在我写完公众号，丢给李笔，他自动帮我改写成 Twitter 线程，连标签都带好了。

**AI 工程师——我的技术后盾**

我虽然写公众号，但很多技术细节其实不太懂。有时候想做个自动化脚本、优化个工作流，纯靠我自己根本搞不定。

现在我有问题就直接问 AI 工程师，他帮我写代码、调试，比我自己折腾快多了。

## 三、真实效果：省了多少时间？

说几个具体的数据：

### 3.1 选题：每天省 30 分钟以上

**以前：** 每天早上要刷一圈 AI 新闻（Twitter、Newsletter、公众号），看到好的选题还要记下来。
**现在：** 9 点醒来，内容总监已经推了 10-15 个选题在等我了。

**效果：每天省 30 分钟以上，全年省 180+ 小时。**

### 3.2 公众号发布：每篇省 20 分钟

**以前：** 写完文章 → 手动转 HTML → 找配图 → 生成封面 → 复制到微信后台 → 预览 → 调整 → 发布。
**现在：** 写完文章 → AI 自动转 HTML、配图、生成封面 → 一键推到草稿箱 → 我预览一下点发布。

**效果：每篇省 20 分钟。**

### 3.3 Twitter 运营：每篇省 15 分钟

**以前：** 公众号发完 → 手动改写成 Twitter 格式 → 拆成多条 → 一个个发。
**现在：** 公众号发完 → 丢给李笔 → 自动改写成线程 → 我审核一下 → 一键发布。

**效果：每篇省 15 分钟。**

### 3.4 记忆不再丢失：省了无数次「重新解释」

这是最爽的一点。

以前每次换对话 session，AI 都会「失忆」，我要重新解释一遍我是谁、我的写作风格是什么、之前踩过什么坑。

现在有了记忆系统，AI 永远知道我是谁。之前踩过的坑（比如「发错过账号」），它比我还记得牢。

**效果：无法量化，但用过的人都懂。**

## 四、怎么做到的？核心技能拆解

### 4.1 记忆系统（ePro-Memory）

这是我最满意的改进。OpenClaw 本身有记忆功能，但用久了会发现问题：它记不住重要的、记住不重要的。

我自己搭了一套分层记忆：

**第一层：MEMORY.md（核心记忆，100 行硬限）**
- 最重要的事：我是谁、写作风格、禁止事项
- 每次对话自动加载

**第二层：memory/（按分类存的素材）**
- 用户个人信息
- 偏好习惯
- 踩过的坑
- 工作经验

**第三层：memory/素材库.md（外部知识）**
- 看到好的文章存下来
- 好的选题方向
- 金句、思想库

**效果：** 现在每次对话，AI 都知道我是谁、喜欢什么风格、踩过什么坑。再也不会「失忆」了。

### 4.2 内容工厂工作流

我用 Obsidian + GitHub 搭了一套内容工厂：

```
服务器写稿 → 自动推送到 GitHub → 本地 Obsidian 同步
```

这样我可以在服务器上写，也可以本地写，两个地方会自动同步。

写完后 AI 自动：
1. 转 HTML 格式
2. 配图（用简笔插画风格）
3. 生成封面
4. 推到微信草稿箱

### 4.3 多 Agent 协作

每个 AI 角色有独立的人格和职责，通过 `sessions_send` 互相发消息。

比如我让李笔改写 Twitter：
```
sessions_send(agentId="libi", message="把这篇公众号改写成 Twitter 线程")
```

他就自动去做了，做完回报给我。

### 4.4 发布检查清单

发文章前必须检查：
1. 确认发哪个号（主号 / 实验室）
2. 检查标题是否合适
3. 删除备用标题
4. 封面提示词要简洁
5. 用 md2html-pro.js 或 simple-md2html.js 转 HTML

## 五、踩过的坑（真诚分享）

### 5.1 模型配得太多

**坑：** 一开始我配了 Kimi、Claude、GPT、MiniMax、智谱... 能配的都配了，觉得越多越好。

**结果：** 根本用不过来，而且每次选模型要纠结半天。

**教训：** 保留 2-3 个稳定的就够了。我的最终选择是：
- MiniMax M2.5：主力模型，性价比高
- 智谱 GLM-4-Coding：写代码专用
- Claude Sonnet 4.6：偶尔用来做复杂推理

### 5.2 记忆总是丢失

**坑：** 一开始没重视记忆系统，每次换 session 都要重新解释一遍我是谁、我的写作风格。

**结果：** 每次都要重复同样的废话，效率很低。

**教训：** 一定要搭自己的记忆层，别全靠 AI 自己发挥。现在我的记忆系统比 AI 自己记得还牢。

### 5.3 发的账号搞混过

**坑：** 有一次应该发主号，结果发到实验室号去了。虽然最后要回来了，但吓出一身冷汗。

**教训：** 发文章前必须确认账号。发错比不发更糟。

### 5.4 封面提示词太复杂

**坑：** 有次想生成一个复杂的封面，写了一堆详细提示词，结果超时了。

**教训：** 封面提示词一定要简洁，越简单越好。复杂的画面交给专业工具。

### 5.5 上下文压缩导致指令丢失

**坑：** OpenClaw 有上下文压缩机制，有时候聊太多了，AI 会「忘了」之前说的重要指令。

**教训：** 重要的指令一定要写在 MEMORY.md 里，不要依赖对话上下文。

## 六、花了多少钱？

这是很多人关心的。

### 6.1 硬件

我现在用的是云服务器（VM），不是本地部署。每月费用大概 100 块左右。

如果你想本地部署，一台 Mac Mini 足够了。

### 6.2 模型费用

我现在用的模型主要是 **MiniMax M2.5**，便宜量大。偶尔用用 **Claude Sonnet 4.6** 写代码，或者 **智谱 GLM-4-Coding** 处理技术问题。

每个月 Token 费用大概 **200-300 块**。

### 6.3 合计

| 项目 | 费用 |
|-----|------|
| 云服务器 | ~100/月 |
| 模型 API | ~200-300/月 |
| **合计** | **300-400/月** |

比请一个真人助理便宜太多了。而且 AI 可以 24 小时待命，不会请假，不会抱怨。

## 八、技术实现细节

### 8.1 部署方式

- 服务器：云服务器（VM）
- 系统：Linux
- 部署方式：Docker 或直接安装

### 8.2 数据流

```
Telegram 发送消息
    ↓
OpenClaw 接收
    ↓
根据 agentId 路由到对应 Agent
    ↓
Agent 处理（读记忆、调用模型）
    ↓
返回结果
    ↓
通过 Telegram 发送给我
```

### 8.3 记忆持久化

```
每次对话结束 → 提取重要信息
    ↓
写入 memory/ 对应文件
    ↓
每 10 分钟 → GitHub 自动同步
```

### 8.4 安全措施

- API Key 不提交到 GitHub（脱敏后备份）
- 敏感记忆不外泄
- 重要操作需要确认（比如发布文章）
