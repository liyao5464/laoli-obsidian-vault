# 用Openclaw+Obsidian搭建内容工厂，写100W+阅读爆文，单篇6000元

## 核心逻辑

工业化内容生产的底层逻辑：
- 抓取细分赛道高赞爆款内容 → 分析内容类型/情绪爆点/文章结构
- 提炼100个痛点选题 → 固定写作风格和提示词框架 → 批量生成

## 架构设计

**Obsidian** = 本地知识数据库（双向链接，持久化保存）
**OpenClaw** = 7x24后台执行引擎

### 软链接打通多Agent
```
实体库：~/Documents/bgggcontent
软链接到各Agent workspace → 共享同一份素材库
```

### 目录结构
- 01-灵感与素材库（日常灵感剪报 + 爆款素材片段）
- 02-选题池/待写选题库
- 03-内容工厂（大纲→初稿→终稿）
- 04-已发布归档

### 强制YAML元数据
每个文件顶部必须有：标题、阶段、关联参考、发布平台

## 四个核心配置文件

1. **SOUL.md** - 安全红线（禁用mv/rm，必须用obsidian-cli）
2. **USER.md** - 写作风格（拒绝AI词汇，短句，真诚极客风）
3. **AGENTS.md** - 启动必读列表（加入SOP读取）
4. **SOP_GZH.md** - 标准作业程序（5个阶段触发条件+动作）

## 金句

> 你维护的不再是几百篇散落的文章，而是一台7x24小时不间断吸收外部信息、不断自我强化的内容处理引擎。
