## 核心观点

---

## 问题

很多新手在配置小龙虾（OpenClaw）的时候，最大的问题不是模型本身，而是自己去手改配置文件。

尤其是 `models` 这一段 JSON，只要多一个逗号、少一个冒号、缩进乱了，gateway 启动直接报错。报的还是那种"有 / 或者有 : 的异常"，看起来像是路径问题，其实大概率是 **JSON 语法错了**。

---

## 解决路径

### 第一步：用官方默认模型跑通

先去**买一个几十块钱的套餐**，一路 `Yes、Yes、Yes`，需要什么 API key 就填什么。**不要动配置文件，不要自己写 JSON，全部走向导。**

先确认小龙虾是能正常跑起来的。

### 第二步：配置其他模型

等跑通了之后，再去配置 AIGoCode 的 Claude Opus 4.6。

而且**这一步也不建议自己改文件**。正确姿势是：把 AIGoCode 的配置文章直接丢给你的小龙虾，让它帮你生成和修改配置。它比你手敲 JSON 稳定得多。

### 第三步：改 API key

很多人卡在 API key 改不动，其实也是因为 JSON 结构被改坏了。只要结构没问题，改 key 是非常简单的事情。

---

## 核心原则

> **不要一上来就折腾配置文件。**

---

## 进阶：多模型、多实例

### 经验之谈

- 一个实例跑 AIGoCode Claude
- 一个实例跑 Codex
- 还有其他模型备用

哪个模型出问题，直接切换。觉得不够聪明就换，限流了就换，报错了就换。**不要死磕一个模型。**

### 资源占用

小龙虾本身内存占用并不高，大概 **500MB** 左右可以跑一个实例。但用得越久、上下文越大，占用会慢慢涨。

64G 内存的 Windows 机器，跑了 10 个实例，大概吃掉 **六七G**，很稳。

---

## 总结

1. 先用 MiniMax 或智谱跑通
2. 不要手改配置文件
3. 用小龙虾帮你生成配置
4. 配多个模型随时切换

**这样走，基本不会卡。**
